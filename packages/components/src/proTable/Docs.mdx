# ProTable

基于 ElementPlus 组件库的 Table 组件封装的高级 Table 组件，除了符合 HIS 的 UI 规范外，还新增了页码、可编辑表格、右键展示菜单、单行选中等功能，同时支持和 `ProForm` 一样的 `valueType` 扩展，在可编辑表格的场景能直接展示对应的表单组件。

## Instance method

表格实例上的方案，由于表格分普通和编辑两种，有些方法只有在编辑表格才有效。下面的文档中会说明编辑表格场景

```tsx
type ProBaseTableInstance<T extends ProComponentObject> = {
  /** 可控制表格的 loading 显示 */
  setLoading: (v: boolean) => void
  /** 获取基础表格的数据，如果是编辑表格，获取到的数据可能不是编辑后的数据 */
  getData: () => T[]
  /** 开启复选框后，获取选中的行数据 */
  getSelectionRows: () => T[]
  /** 获取基础表格带内部 key 的数据，很多场景需要用到内部 key */
  getDataHasKey: () => T[]
  /** 和 getDataHasKey 的区别在于，该方法会清除空行的数据 */
  getRealDataHasKey: () => T[]
  /** 获取表格数据的长度 */
  getTableDataSize: () => number
  /** 刷新表格，参数若为 true，则会清除搜索条件和页码，若 refreshClearChecked 为 true，还会清除行选中态 */
  refresh: (v?: boolean) => Promise<ProComponentAny>
  /** 重新计算表格高度和宽度 */
  resize: () => void
  /** 执行表格的搜索 */
  onSearch: (params?: ProComponentObject) => Promise<ProComponentAny>
  /** 和 onSearch 区别在于，该方法不会清除页码 */
  onCurrentSearch: (params?: ProComponentObject) => void
  /** 设置表格搜索的参数 */
  setSearchForm: (data: ProComponentObject) => void
  /** 获取表格搜索的参数 */
  getSearchForm: () => ProComponentObject
  /** 设置表格数据，第二个参数为 true，表示覆盖表格内的值，否则是追加 */
  setData: (data: T[], isCover?: boolean) => void
  /** 设置部分表格的行数据，此时的 data 需要带内部 key，该方法会根据 key 去匹配覆盖表格内对应的行数据 */
  setDataClad: (data: T | T[]) => void
  /** 设置表格的行选中 */
  setCurrentRow: (row: T) => void
  /** 根据传入的行数据，判断是否为表格本地新增的数据，是本地新增的行返回 true */
  getRowIsLocal: (row: T) => boolean
  /** 对外提供的操作方法集合 */
  actions: {
    /** 导出成 excel */
    exportMethod: (options?: ProTableToolbarExportProps) => Promise<ProComponentAny>
    /** 导入 excel */
    importMethod: (file: Blob, options?: ProTableToolbarImportProps<T>) => Promise<ProComponentAny>
    /** 编辑表格中，新增一行 */
    add: (rowData: T) => void
    /** 编辑表格中，删除行 */
    delete: (rowData: T) => void
  }
  /** 编辑表格中，获取的所有表单控件实例 */
  getFormInstances: () => Record<string, ProComponentAny>[]
  /** 编辑表格中，根据 key 返回对应的表单控件实例 */
  getFormInstanceByKey: (key: string, rowData: T) => ProComponentAny
  /** 编辑表格中，校验数据的方法 */
  validate: (cb?: ProTableFormValidateType) => Promise<boolean>
  /** 编辑表格中，获取对应的表格数据，此时数据是编辑后的 */
  getFormData: () => T[] | undefined
  /** 编辑表格中，获取修改的数据，包含新增，修改和删除，格式{ add: T[], edit: T[], remove: T[] } */
  getFormRecord: () => ProRecordDataType<T>
  /** 等同于 setData(data, true) */
  resetForm: (data: T[]) => void
}
```

### 基础用法

ProTable 的基本用法，**`rowKey` 是必须配置的，不能有重复**。`valueType` 默认为文本框，和 `ProForm` 文档介绍的 `valueType` 一致。下面演示的就是一个常见的表格展示，包含搜索，新增，修改和停用。

<Canvas of={BasicStories.Basic} />

从结构上看，一个常规的表格包含三部分头部包含文字描述、搜索区和按钮、中间是表格内容、底部是分页。

#### 头部

头部使用 `toolbar` 来配置，其中包含以下属性

```tsx
type ProTableToolbarProps<T> = {
  className?: string
  title?: string
  filtersCustom?: (v: ProTableToolSlotParams) => VNode | VNode[]
  searchCustom?: (v: ProTableToolSlotParams) => VNode | VNode[]
  buttonsCustom?: (v: ProTableToolSlotParams) => VNode | VNode[]
  custom?: boolean
  export?: boolean | ProTableToolbarExportProps
  import?: ProTableToolbarImportProps<T>
}
```

- className，toolbar 区域的自定义样式
- title，表格标题
- filtersCustom，返回筛选区内容的函数
- searchCustom， 返回搜索区内容的函数
- buttonsCustom，返回按钮区内容的函数
- custom，是否展示“表格自定义列”按钮
- export，是否展示“导出”按钮
- import，是否展示“导入”按钮

筛选区、搜索区和按钮区的函数入参 `ProTableToolSlotParams` 包含的字段如下

```tsx
type ProTableToolSlotParams = {
  onSearch: (params?: ProComponentObject) => void
  onCurrentSearch: (params?: ProComponentObject) => void
  searchForm: Reactive<ProComponentObject>
}
```

- onSearch，传入参数，执行表格搜索方法，会重置页码
- onCurrentSearch，传入参数，执行表格搜索方法，不重置页码
- searchForm，筛选区和搜索区配置的参数，是 `Reactive` 对象

```tsx
const customBase = () => {
  return (
    <>
      <ActionButton<RowVO>
        buttonProps={{
          type: 'primary',
          label: '新增',
        }}
        dialogProps={formDialog}
        request={(data?: RowVO) => {
          console.log(data, 'request')
        }}
      />
    </>
  )
}
const filterBase: ProTableToolbarProps<RowVO>['filtersCustom'] = ({
  searchForm,
  onSearch,
}) => {
  // 回车事件处理函数
  const handleEnter = (e: KeyboardEvent | Event) => {
    const keyEvent = e as KeyboardEvent
    if (keyEvent.key === 'Enter') {
      onSearch()
    }
  }
  const handleModelV = (v: string) => {
    searchForm.text = v
  }
  return (
    <ElInput
      modelValue={searchForm.text}
      onUpdate:modelValue={handleModelV}
      placeholder="请输入文本"
      onKeydown={handleEnter}
    />
  )
}
const toolbarColumns = {
  title: '基础表单',
  filtersCustom: filterBase,
  buttonsCustom: customBase,
}
```

上面的代码中能看到详细的使用，这里说明一下按钮为什么没有使用 `ElButton`。如果按钮的操作是改变表格数据，推荐使用 `ActionButton` 组件，该组件内置了 `ProDialog`的功能，同时还集成了表格刷新的处理。

如果你喜欢 vue 的 template 的写法，`filtersCustom`, `searchCustom`, `buttonsCustom` 也有对应的 slot name 是 `toolbar-filters`, `toolbar-search`, `toolbar-buttons`。

#### 表格内容

表格主体核心就是数据和 `columns` 配置。常规业务场景的表格数据都是从 API 获取，当前 demo 中使用的是 `request` 模拟请求表格数据，其他获取表格数据的方式放在 `Data` 章节详细介绍。

`columns` 配置，单个 column 包含以下属性

```tsx
type ProColumn<T extends ProComponentObject> = {
  /** 是否缩略 */
  ellipsis?: boolean | { showTitle?: boolean }
  /** 是否拷贝 */
  copyable?: boolean
  /** 列标题旁边的 tooltip */
  tooltip?: string
  /** 列标题 */
  label?: string
  /** 字段名 */
  prop?: string
  /** 类型，支持 seq(序号) */
  type?: string
  /** 对齐方式 */
  align?: 'center' | 'right'
  /** column 模式 */
  mode?: string
  /** 控件类型 */
  valueType?: string
  /** 控件 props */
  proFieldProps?: ProComponentObject
  /** 是否必填 */
  required?: boolean
  /** 控件 校验规则 */
  rules?:
    | PropFormFieldProps['rules']
    | ((rowIndex: string, rowData: T) => PropFormFieldProps['rules'])
  /** 控件联动规则 */
  effects?: (rowIndex: string) => PropFormFieldProps['effects']
  /** cell自定义渲染 */
  render?: (rowData: T, rowIndex: number) => VNode
  /** 在 table 中隐藏 */
  hideInTable?: boolean
  /**
   *  @name 可编辑表格是否可编辑
   *
   * @example 不允许编辑
   * editable={false}
   *
   * @example 如果id=1不允许编辑
   * editable={(params)=> params.rowData.id !==1 }
   */
  editable?: false | ((params: ProCellRendererParams<T>) => boolean)
  /** cell自定义渲染 */
  cellRenderer?: ProCellRenderer<T>
  /** 编辑状态cell自定义渲染 */
  editCellRenderer?: ProCellRenderer<T>
  /** 表头自定义渲染 */
  headerCellRenderer?: ProHeaderCellRenderer
  /** 表头支持全选 */
  headerIsAll?: boolean
  /** 嵌套表头 */
  children?: ProColumn<T>[]
  /** 是否校验通过才能触发回车 */
  validateBeforeEnter?: boolean
  /** actionButton组件 */
  actions?: ActionButtonProps<T>[]
}
```

- label，列的标题，配置 `title` 效果一致，但推荐使用 `label`
- prop，列的字段名，配置 `field` 和 `dataKey` 效果一致，但推荐使用 `prop`
- type，特殊的列，支持 `index` 和 `seq`，都是显示内置的序号
- hideInTable，表格列中隐藏
- align，表格内容对齐的方式，支持 `center` 和 `right`
- mode，支持 `read` 和 `edit`，在可编辑表格场景中，配置了 `read` 表示当前列不能编辑，优先级大于 `editable`
- valueType，控件类型，同 `ProForm` 的 `valueType`
- proFieldProps，控件 props，同 `ProForm` 的 `fieldProps`
- required，是否必填，开启后列标题会有 \*，必填在编辑表格场景才生效
- rules，校验规则，同 `ProForm` 的 `rules`
- effects，联动规则，同 `ProForm` 的 `effects`
- render，单元格自定义渲染，传入两个参数
- editable，单元格是否可编辑，支持函数方式。优先级低于 `mode`
- cellRenderer，单元格自定义渲染，和 `render` 的区别在于参数不一致
- editCellRenderer，编辑状态单元格自定义渲染
- headerCellRenderer，表头自定义渲染
- headerIsAll，列表头支持全选，此时列的标题只展示复选框
- validateBeforeEnter，否校验通过才能触发回车，默认 `false`
- actions，操作列的按钮组件的配置
- children，嵌套表头的场景
- ellipsis，单元格文字超出是否缩略
- copyable，单元格文字是否支持拷贝
- tooltip，列标题旁边的提示

#### 分页

分页配置 `pagination` 是在 element-plus 组件库提供的 `ElPagination` 组件配置的[基础](https://element-plus.org/zh-CN/component/pagination#%E5%B1%9E%E6%80%A7)上增加了 `position` 和 `onChange`。默认是显示分页组件的，如需隐藏设置成 `false` 即可。

- position，分页组件在表格中的位置，可配置 'topLeft', 'topCenter', 'topRight', 'bottomLeft', 'bottomCenter', 'bottomRight'，默认 `bottomRight`
- onChange，分页切换和页码切换触发的回调函数，包含两个参数， currentPage 和 pageSize。

### 表格获取数据

ProTable 提供了几种获取数据的方法，以下 props 都和获取数据相关。

- request，请求数据，完成后会自动把数据填入到表格中
- params，request 的参数，修改之后会触发 request 请求
- postData，处理数据的函数，若需要处理 request 方法请求到的数据，配置该函数
- defaultData 和 dataSource，初始化表格数据，后期变化不会影响表格内部的数据。两个属性效果一致，推荐使用 defaultData
- data，动态数据源，可控制表格数据变化，若要保持数据一致，需要配合 `onDataChange` 方法同步数据
- onDataChange，表格数据任何变化都会执行的回调函数

<Canvas of={BasicStories.Data} />

一般使用 `request` 方法获取表格数据，若要实现联动，可配置 `params`。若要在初始化时传入数据，使用 `defaultData`，若要在表格外控制表格的数据，使用 `data` 和 `onDataChange`。ProTable 内部会给数据添加一个私有 `_proKey` 的 key，**仅供组件内部使用，外部切勿使用**。

#### 单击行选中

- firstRowSelected，初始化选中第一行，若数据是异步获取，则在数据获取到选中第一行
- currentChangeRepeat，配置 current-change 单击同一行可重复触发，默认 false
- currentSelectedRow，current-change 的默认值，和 firstRowSelected 互斥，表格外部可使用该属性控制当前选中哪一行，一般用在下拉表格场景
- current-change，单击行事件回调函数，该函数会接收两个参数，第一个是当前选中行数据，第二个是上一次选中行的数据

很多时候业务中需要在表格初始化时就要选中第一行，可使用 `firstRowSelected`，同时也会触发 `current-change` 回调，此函数中能获取到当前选中的行数据。

配置 `menuConfig` 属性，具体字段如下

### 布局

ProTable 提供以下 props 方便开发者去修改样式。

- containerClass，外层容器类名
- tableClassName，同 containerClass 效果
- bodyClassName，table 中 body 类名
- bodyMainClassName，table 中 main 类名

<Canvas of={BasicStories.Layout} />

ProTable 提供了一些高度相关的配置

- height，表格高度，要包含表格头部和页码高度，内部会自动计算去除这两部分高度
- maxHeight，表格最大高度，只有开启 `autoHeight` 的场景才生效
- autoHeight，是否开启自适应高度，开启后，表格的高度不在撑满外层容器，同时 `height` 值失效
- sameMaxHeight，开启后，`height` 和 `maxHeight` 都使用 `height`，`maxHeight` 值失效

正常来说什么都不用传，表格默认同外层容器高度一致。

在编辑表格的场景，需要高度自适应，添加一行就撑开一行，但是也需要达到一定高度时出现滚动条，此时需要同时配置 autoHeight = true 和 maxHeight = 500(任意正数)。autoHeight 开启后，sameMaxHeight 就强制关闭，这两个属性互斥。

sameMaxHeight 为了让 `element-plus` 的 `el-table` 的 height 和 maxHeight 一致，此时 ProTable 传入的 maxHeight 是无作用的。

### 多选

由于 `el-table` 的多选功能不支持受控，ProTable 自行实现了多选逻辑。配置 `rowSelection` 即可实现多选

```tsx
type ProTableRowSelectionProps<T> = {
  /** 目前只支持 checkbox */
  type?: 'checkbox'
  /** 互斥，表格中只能选中一个复选框 */
  repel?: boolean
  /** 树形表格下多选的配置，是否取消和父级关联，默认 false 不取消 */
  checkStrictly?: boolean
  /** 属性表格下多选的配置，子级是否展示 checkbox，默认 false 不展示 */
  childrenCheckbox?: boolean
  /** 受控属性，配置即可控制表格选中的行 */
  selectedRowKeys?: KeyType | KeyType[]
  /** 是否隐藏全选，默认不隐藏 */
  hideSelectAll?: boolean
  /** 是否支持单行点击选中，默认不支持 */
  rowClick?: boolean
  /** 当前行复选框是否可选中 */
  selectable?: (rowData: T) => boolean
  /** 多选发生变化时触发的回调 */
  onChange?: (selectedRowKeys: KeyType[], selectedRows: T[]) => void
  /** 单击复选框时触发的事件，若返回 false，则不能选中 */
  onSelect?: (selectedRows: T[], row: T) => void | boolean | Promise<boolean>
}
```

```tsx
  /** 操作列属性配置 */
  optionColumnProps?: ProComponentObject
}
```

- type，编辑类型，single 是每次只有一行进入编辑态，multiple 是支持多行同时编辑，cell 是只有单个单元格在编辑态，only 整个表格都是编辑态。
- mode，支持编辑场景切换到只读态
- defaultRow，业务场景需要进入页面就新增一行空行，方便使用者直接输入，默认为 true
- keyboard，单元格编辑时，回车进入下一个单元格

**当 type 配置了 cell 时，表格中是不会内置操作列的，需要开发者自行配置操作列**，此时和操作列相关的属性无效。删除方法可通过实例上的方法去删除。

```javascript
{
  title: '操作',
  width: '80',
  valueType: 'option',
  fixed: 'right',
  render(rowData) {
    return (
      <>
        <ElButton link type="danger" onClick={() => tableEditorRef.value?.actions.delete(rowData)}>
          删除
        </ElButton>
      </>
    )
  },
}
```
